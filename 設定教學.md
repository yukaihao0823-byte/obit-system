# 🕊️ 電子訃聞系統 — 完整部署教學

-----

## 📁 檔案清單

```
obit-system/
├── index.html          ← 登入頁
├── admin.html          ← 後台管理（員工使用）
├── obituary.html       ← 訃聞展示頁（家屬使用）
└── firebase-config.js  ← 🔧 你需要填入設定的地方
```

-----

## 第一步：申請 Firebase（約 10 分鐘）

1. 前往 **https://console.firebase.google.com**
1. 登入 Google 帳號，點「建立專案」
1. 專案名稱填：`obit-system`（或自訂）
1. Google Analytics 可以關閉，按「建立專案」

### 啟用 Authentication（登入功能）

1. 左側選單 → **Authentication** → 「開始使用」
1. 點「Sign-in method」分頁
1. 啟用 **電子郵件/密碼**（點進去開啟第一個開關）
1. 啟用 **Google**（點進去開啟，填入你的公司 email 為支援信箱）

### 啟用 Firestore Database（資料庫）

1. 左側選單 → **Firestore Database** → 「建立資料庫」
1. 選「以生產模式啟動」→ 選擇位置 **asia-east1（台灣）** → 完成

### 啟用 Storage（圖片儲存）

1. 左側選單 → **Storage** → 「開始使用」
1. 選「以生產模式啟動」→ 選 **asia-east1** → 完成

### 設定安全規則 — Firestore

1. Firestore → 「規則」分頁，貼上以下內容：

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // 案件資料：登入用戶可讀寫
    match /cases/{caseId} {
      allow read: if true;  // 家屬不需登入即可讀取訃聞
      allow write: if request.auth != null
        && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role
           in ['admin', 'staff'];
    }

    // 用戶資料：本人可讀，admin 可寫
    match /users/{userId} {
      allow read: if request.auth != null && request.auth.uid == userId;
      allow write: if request.auth != null
        && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
      allow create: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

1. 按「發布」

### 設定安全規則 — Storage

1. Storage → 「規則」分頁，貼上：

```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /photos/{caseId} {
      allow read: if true;
      allow write: if request.auth != null;
    }
  }
}
```

1. 按「發布」

## 第二步：取得 Firebase 設定碼（2 分鐘）

1. Firebase Console → 左上角齒輪 ⚙️ → 「專案設定」
1. 往下捲到「你的應用程式」
1. 點「</> 網頁」圖示，輸入暱稱 `obit-web`，按「註冊應用程式」
1. 你會看到類似這樣的程式碼：

```javascript
const firebaseConfig = {
  apiKey: "AIzaSy...",
  authDomain: "obit-system-xxxx.firebaseapp.com",
  projectId: "obit-system-xxxx",
  storageBucket: "obit-system-xxxx.appspot.com",
  messagingSenderId: "123456789",
  appId: "1:123456789:web:abcdef"
};
```

1. 打開 **firebase-config.js**，把每個值填進去
1. 同時修改：
- `COMPANY_NAME` → 改成你公司的名稱
- `OBITUARY_BASE_URL` → 等 Netlify 部署後再填（見第四步）

-----

## 第三步：部署到 Netlify（5 分鐘）

1. 前往 **https://netlify.com** → 免費註冊
1. 登入後，把整個 `obit-system` 資料夾**直接拖進**網頁中間的拖放區
1. 等待 30 秒，Netlify 會給你一個網址，例如：
   `https://cheerful-sundae-abc123.netlify.app`

### 設定自訂網域（選用）

- Netlify → 「Domain settings」→ 可綁定你自己的網域
- 例如：`obit.yourcompany.com`

-----

## 第四步：更新分享網址設定

1. 打開 `firebase-config.js`
1. 把 `OBITUARY_BASE_URL` 改為：
   
   ```javascript
   const OBITUARY_BASE_URL = "https://你的netlify網址/obituary.html";
   ```
1. **重新拖上傳**一次到 Netlify（覆蓋舊版本）

-----

## 第五步：建立第一個管理員帳號（5 分鐘）

### 方法：Email + 密碼

1. Firebase Console → **Authentication** → Users 分頁
1. 點「新增使用者」
1. 填入員工 Email 與初始密碼
1. 複製產生的 **User UID**（一串字母數字）

### 設定角色

1. Firebase Console → **Firestore Database**
1. 點「開始收藏」→ 輸入 `users`
1. 文件 ID 填入剛才複製的 **User UID**
1. 新增欄位：
- `email`：填員工 email（字串）
- `name`：填員工姓名（字串）
- `role`：填 `admin`（字串）

### 第一位員工以後的帳號

- 新員工用 Google 登入後，系統會顯示「帳號審核中」
- 管理員到 Firestore → `users` 集合找到該員工 → 把 `role` 改成 `staff`（一般員工）或 `admin`（管理員）

## 日常使用流程

```
1. 員工登入 → index.html
      ↓
2. 後台新增案件 → 填寫資料、上傳遺照、選款式
      ↓
3. 按「發布案件」
      ↓
4. 複製分享連結 or 發送 LINE 分享
      ↓
5. 家屬手機點開連結 → obituary.html?id=OB-XXXXX
```

## 常見問題

**Q：家屬打開連結看不到內容？**

- 確認案件已「發布」（非草稿狀態）
- 確認 `OBITUARY_BASE_URL` 已填入正確網址

**Q：Google 登入跳出錯誤？**

- Firebase Console → Authentication → Sign-in method → Google → 確認已啟用
- 確認 Netlify 網址有加入「已授權的網域」：Firebase → Authentication → Settings → Authorized domains → 新增你的 Netlify 網址

**Q：照片上傳失敗？**

- 確認 Firebase Storage 規則已發布
- 照片檔案建議壓縮至 2MB 以下

**Q：如何讓員工重設密碼？**

- 員工在登入頁輸入 Email，點「忘記密碼」即可收到重設信
